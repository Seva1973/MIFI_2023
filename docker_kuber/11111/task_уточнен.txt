нужно развернуть кластер Minikube на базе двух контейнеров docker (В качестве основных образов для docker - контейнеров нужно взять публичные образы:

mysql:8.0  в качестве базы данных,
apache 2.4 в качестве веб-сервера 

всю систему развернуть в ОС Debian 11. Для этого нужно создать файлы:
1) ConfigMap.yaml где необходимо прописать путь до базы данных mysql для нашего веб-сервера, чтобы он мог обращаться к ней. в ConfigMap нужно создать переменную, которая указывает на Service базы данных.
2) mysql-secret.yaml где нужно указать пароль и логин для подключения к БД. Помним, что в этом файле данные должны храниться в base64
3) mysql-deployment.yaml - где необходимо добавить раздел serviсe: имя сервиса , а также селектор (он обязательно должен совпадать с названием пода, иначе Selector не поймёт куда перенаправлять все пакеты). Дальше указываем ports: - protocol; у нас TCP. Port — это любой порт на котором будет принимать запросы Service, а targetPort: — это место, куда будут перенаправляться запросы (он обязательно должен совпадать с портом, на котором работает pod). 
4) webapp-deployment.yaml - деплоймент для web приложения где также написать service имя сервиса , а также селектор (он обязательно должен совпадать с названием пода, иначе Selector не поймёт куда перенаправлять все пакеты). Дальше указываем ports: - protocol; у нас TCP. Port — это любой порт на котором будет принимать запросы Service, а targetPort: — это место, куда будут перенаправляться запросы (он обязательно должен совпадать с портом, на котором работает pod). 
5) файл index.php нужно переписать, чтобы адрес до БД и логин/пароль были прописаны в переменных окружения
6) обязательное условие образ веб-приложения должен брать данные для подключения к БД из переменных окружения
7) необходимо сделать таблицу (в формате Excel) с инвентаризацией вашего кластера. В кратком виде указать установленное ПО на нодах, количество нод, ОС на нодах, адреса.


В minikube развернут под с базой данных mySQL с названием world. Необходимо самое простое решение для отображения одной из 3 таблиц этой базы в web-браузере, так, чтобы запуская minikube service webapp-service web браузер выводил данные таблицы
