1) Прототип
На хостовой машине устанволена MySQL 8.0 и PHP_Apache для создания прототипа базы данных
В БД загружена база данных world.sql
Создан файл mysql_apache.php для связи сервера и БД
Выведены данные БД в браузер командой в строке http://10.0.0.1/mysql_apache.php

Этот проект был взят в качестве прототипа для развертывания в кластере Миникуб

2) Миникуб
На хостовой машине установлен Minikube с драйвером Docker
Были созданы файлы конфигурации БД и секрета БД: mysql-config.yaml & mysql-secret.yaml (в последнем имя доступа и пароль были указаны в закодированном виде в base 64)
Был создан файл для развертывания контейнера в кластере с БД MySQL, куда после развертывания была скопирована БД world.sql (kubectl cp)
Был создан файл для развертывания пода с локальным веб-сервером php-apache-mysql.php, где данные для доступа были указаны в явном виде - файл был нужен для теста работоспособности контейнеров внутри кластера
Был создан тестовый под для развертывания локального веб-сервера без всяких зависмостей webapp-pod.yaml, куда после развертывания был скопирован php-apache-mysql.php в var/www/html
Внутри пода webapp-pod.yaml была выполнена команда php webapp-pod.yaml - связь с целевой БД и целевой таблицей была установлена
Был создан файл webapp-deployment.yaml для развертывания полноценного приложения веб сервера, который был должен самостоятельно сформировать файл для связи с БД внутри контейнера, а также были прописаны зависимости и сервисы
Был создан специальный для этого контейнера файл php-apache-mysql-copy.php
После развертывания нового пода в кластере с веб-сервером выявились проблемы:
- Не был скопирован внутрь "пода" сервера php-apache-mysql-copy.php хотя и окружения и все пути были указаны, разрешения для операции над файлом - достаточны
- После ручного копирования php-apache-mysql-copy внутрь контейнера и запуска изнутри через команду php терминал вернул сообщение что отсутствует драйвер mysqli 
- Был переписан webapp-deployment.yaml - куда в явном виде было приписана установка образа с драйвером mysqli (не устанволен, как показал анализ окружения php)
- В образе был устанвлен по-умолчанию драйвер PDO который вроде-как должен был также связываться с базами данных.
- Были переписаны файлы webapp-deployment.yaml и php-apache-mysql-copy.php для применения драйвера PDO. Деплоймент был добавлен в контейнер (apply) а файл *.php - просто скопирован. Но, попытка запуска никчему ни привела. Опять терминал вернул сообщения что нет устанвленного драйвера
- Не привела к успеху и запуск совершенно простого, такого же как я применял в тестовом поде, где в явном виде были прописаны для БД адреса, порты, имя доступа и пароли, что выявило проблему непосредственно в развернутом контейнере с локальным веб-сервером, что удивительно, так как в тестовом поде веб севрера (webapp-pod)использовался все тот же образ  php:7.4-apache
